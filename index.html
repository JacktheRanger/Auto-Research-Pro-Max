Research Preview

An autonomous workbench that takes you from initial idea to a complete paper manuscript. Provide relevant literature, review AI-generated plans, and let the AI autonomously design and validate experiments in a local sandbox with just one click.

Comprehensive Research Pipeline

Mandatory Brief Intake

Before execution, outline your title, idea, background, direction, goals, constraints, and must-read papers. No implicit black-box prompting.

Hybrid Grounding Engine

Upload local PDFs and provide remote paper URLs simultaneously. We enrich metadata (DOI, venue, authors) and automatically chunk/embed for retrieval.

Plan-Gated Execution

The system drafts an experiment and execution plan first. It must pass an explicit human approval gate before moving to sandbox run.

Repository-Aware Sandboxing

Attach a local repo path or Git URL alongside setup/run commands. The sandbox executes real-world workloads, tracks timeouts, and collects artifacts.

Live Pipeline Tracking

Monitor the real-time progress of 15+ sub-stages via WebSocket updates, pause, resume, or rollback when an approval gate is reached.

Complete Deliverables

Export true Markdown, LaTeX, BibTeX files, compiled PDF manuscripts, and a full zipped delivery bundle containing all research products.

How It Works

1

Configure Settings

Set your OpenAI/Codex endpoints, keys, and preferred models right from the web GUI.

2

Create Definition

Input your research hypothesis, compute budget, API constraints, and artifact expectations.

3

Literature Grounding

Import paper links, PDFs and perform live discovery. The engine parses them contextually.

4

Approve Strategy

Review the system-generated strategy. Reject or approve to unblock the execution pipeline.

5

Live Observation

Watch stages transition from pending to running, inspect internal metadata, or rollback on failure.

6

Evaluate Results

Gather the evaluated experiment code and download the final LaTeX manuscript with peer-review rubrics.

Privacy & Local-First

Your research context belongs to you. Auto Research Pro Max is built with local isolation natively in mind.

Local Storage

DB state, plans, running outputs, and artifacts are all stored in the local SQLite database.

Offline Fallback

If no API key is provided, the UI functions perfectly using deterministic fallback text.

Private PDFs

Uploaded papers are saved strictly to the local disk. No opaque third-party cloud hosting.

Data Control

All data egress is strictly confined to user-configured APIs and endpoint URIs you explicitly defined.
